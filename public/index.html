Getting Started

Setup

You'll need to open a terminal, change directory into the may-challenges directory and run npm install

Once that has completed you'll need to then run node server.js in order to start serving the assets.

Everything inside the public folder will be served on http://localhost:8080.

Before beginning work, create and checkout a new Git branch "challenges" and after every challenge you should commit your work to this branch.

Tools provided

You've been provided with a basic CSS framework. It doesn't do much, but what it does do is documented at the link below.

For the later challenges, there is also an api provided.

See the docs

Challenges

HTML Challenge

The challenge is to create a document using HTML. This document should be a recipe for a food or drink you enjoy.

Your document should be valid, semantic HTML5, which will also work with screen reading software. You should also ensure that your HTML is structured in such a way that the document information flows sensibly.

An empty file challenge.html has been provided in the public folder, and you should put all code here.

You should write no CSS of your own, but instead your document should include the stylesheet base.css found in the public folder. This will provide a very simple framework for laying out your page.

Your document should include at least the following element types, but can contain any extra you require.

  • paragraphs
  • tables
  • image tag
  • classes
  • list

Your HTML Challenge

CSS Challenge

Now that you have a valid html document, you should go back to it and modify it to include the empty stylesheet challenge.css. This is the only change you should make to your HTML.

All edits should now be in the CSS only.

You will be given a design for your recipe which you are to implement. This design will be intended to cover:

  • The CSS Box model
  • Positioning attributes
  • background images
  • Pseudo selectors and pseudo elements

JS Challenge

Congratulations! Your recipe is a huge success! The content and design are so good that you've gone viral. Your employer would now like to capitalise on that traffic by capturing as many visitors as possible into a signup form.

The backend team have made new server endpoints for you, at /analytics and /newsletter. You need to capture peoples name and email address and POST it to this new service.

You've been tasked to complete the following steps, at the end of each step you must commit your work.

  1. Using jquery, on page load send a POST request to /analytics including the URL of the page.
  2. Hardcode a new modal that sits over the existing content and has fields for name and email, as well as a submit button.
  3. Use jQuery to submit those details to the /newsletter endpoint on click of the button
  4. When the details have been submitted, close the modal and allow access to the content behind.
